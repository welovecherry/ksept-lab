## 9~12. LLM 개념

### LLM이란
- **Large Language Model** = 초강력 자동완성기. **다음 토큰**(단어/단어조각)을 앞 내용 보고 예측한다.
- 한 번에 한 토큰씩 생성하고, 매 토큰은 **앞의 모든 것**(내 프롬프트 + 시스템 지시 + 지금까지의 출력)에 좌우된다.
- **컨텍스트가 답을 바꾼다.** 예: "Mary had a little ___" → 보통 `lamb`, "재미있게 끝내줘" 맥락 → `llama...`.

### 잘하는 것 / 못하는 것
- 잘함: 글·코드·요약 생성, 번역, 정보 추출, 이미지 읽기, 단계적 추론.
- 도움 없이는 못함: 이전 대화 기억, 최신 정보, 실제 행동, 코드 실행·도구, 자기 답 검증.
- **환각(hallucination)**: 모르면서도 그럴듯하게 지어냄 → 믿을 자료(컨텍스트)를 주면 품질↑.
- 정리: **LLM = 엔진**, **AI 앱 = 그 엔진으로 신뢰 가능한 제품 만들기**.

### 챗 앱 vs 모델 API (가장 중요)
- 챗 앱(claude.ai 등) = 완성품. 대화 기억·파일·웹검색·UI 다 해줌. 구독 과금. 일반 사용자용.
- 모델 API(Anthropic 등) = 날것 접근. **매 요청마다 전체 히스토리 직접 전송**, 도구 직접 구현, 토큰당 과금. 개발자용.
- 비유: 챗앱 = 완성된 자동차, API = 엔진만 받기(차는 내가 조립). **API로 가면 내가 그 앱의 개발자**가 된다.

# 심화 튜토리얼 (단계별)

> 여기서부터는 슬라이드마다 배경 지식을 단계별로 깊게 정리한다.

## 10. LLM이란? — 단계별로 뜯어보기

### 1단계 · "토큰"이 기본 단위
- LLM은 글자나 단어가 아니라 **토큰**(token) 단위로 읽고 쓴다. 토큰 = 단어 또는 **단어 조각**.
- 예: `unbelievable` → `un` + `believ` + `able` 처럼 쪼개짐. 공백·기호도 토큰이 될 수 있다.
- 왜 조각으로? 세상의 모든 단어를 통째로 외우면 사전이 너무 커진다. **적은 조각으로 무한히 조합**하는 게 효율적이다.
- 감 잡기: 영어 기준 보통 **1토큰 ≈ 0.75단어**. "토큰당 과금"이 여기서 나온다.

### 2단계 · 하는 일은 딱 하나 — "다음 토큰 예측"
- LLM의 본질은 단 하나: **"지금까지의 토큰들"을 보고 "다음 토큰"의 확률을 계산**하는 것.
- 토큰 하나를 뽑으면, 그걸 다시 입력 끝에 붙이고 → 또 다음 토큰을 예측 → 반복. (이걸 **autoregressive**라고 함)
- 비유: **세계 최강 끝말잇기**. 매 순간 "다음에 올 가장 그럴듯한 조각"을 이어 붙인다.

### 3단계 · 어떻게 그렇게 잘 맞히나 (학습)
- 책·웹·코드 등 **수조 개 토큰**으로 "다음 토큰 맞히기"를 무수히 반복 연습한다.
- 그 과정에서 **파라미터**(수천억 개의 숫자 다이얼)를 조금씩 돌려 예측 오차를 줄인다.
- 결과적으로 문법·사실·문체·코드 패턴이 **파라미터 안에 압축**된다. (모델 파일이 곧 이 다이얼 값들)

### 4단계 · 컨텍스트가 답을 바꾼다 (가장 중요)
- 다음 토큰 확률은 **앞에 무엇이 있었는지에 전적으로 의존**한다.
- 같은 빈칸도 맥락 따라 답이 갈린다: `Mary had a little ___` → 보통 `lamb`, 그런데 "재미있게 끝내줘" 맥락 → `llama...`.
- 컨텍스트 = **시스템 지시 + 내 프롬프트 + 지금까지의 출력** 전부.
- 그래서 LLM 활용의 핵심은 **"좋은 컨텍스트를 주는 것"**이다.

### 5단계 · 기억의 한계 — 컨텍스트 윈도우
- 한 번에 볼 수 있는 토큰 수에 **상한(컨텍스트 윈도우)**이 있다.
- 그 창을 넘어간 옛 내용은 모델이 **못 본다** → 그래서 대화 기억은 **매 요청마다 히스토리를 다시 넣어줘야** 한다 (9~12에서 본 API의 특징).

### 6단계 · 매번 조금 다른 이유
- 다음 토큰을 확률 분포에서 **뽑기** 때문에, 같은 질문도 답이 조금씩 달라질 수 있다 (이 무작위성 정도가 `temperature`).
- 이 성질이 **창의성**의 원천이자 **환각**의 원천이기도 하다.

### 한 줄 요약
LLM = **토큰 단위로 다음을 예측하는 거대한 확률 엔진**. 컨텍스트가 예측을 좌우하고, 기억엔 한계(윈도우)가 있으며, 답은 확률적이다.
