# 최고 설정 이식 — 그리드 우승 검색설정 + 균형 프롬프트를 챗봇에 심기

> 🎯 밤샘 그리드가 뽑은 **최고 검색설정**과 오늘 실험한 **균형 프롬프트**를 실제 챗봇(`app.py`)에 적용해 대회 출전본을 만든다.
> 검색 실험 = [`06_30_act1_harness.md`](06_30_act1_harness.md) · 채팅 UI = [`06_30_chat_ui.md`](06_30_chat_ui.md)(다른 세션) · 프롬프트 놀이터 = `harness/try_prompt.py`.

## 사용자 시나리오 (가장 먼저 읽힌다)

- **누가**: 홍정민 — 아침에 밤샘 결과(리더보드)를 받아, *제일 잘 맞히는 설정*을 챗봇에 이식하는 사람.
- **언제**: 그리드가 `runs.jsonl`을 다 채운 직후(≈792줄).
- **어떻게**:
  1. 리더보드를 봐서 **1등 검색설정**(청킹·임베딩·검색방식·K)을 고른다.
  2. 그 설정으로 **인덱스를 다시 굽고**, `app.py`가 그 인덱스·그 검색방식을 쓰게 바꾼다.
  3. 답변 **프롬프트를 균형톤**(친절+간결+그라운딩)으로 바꾼다.
  4. **연습문제 5개**를 채팅에 넣어 답+인용이 좋은지 눈으로 확인한다.
- **결과**: 챗봇이 *옳은 조항을 물어와*(검색) *완전하고 친절하게 인용하며*(생성) 답한다 = **대회 출전본**.
- **잘 안 될 때**: 1등이 2등보다 근소하게 낫고 훨씬 비싸면 → *더 싸고 단순한 쪽* 선택.

> 비유: 밤새 요리 대회 예선을 돌려 **우승 레시피**를 찾았으니, 이제 그 레시피로 *본선 요리*를 실제로 차리는 단계.

## 배경

- 그리드(H3b)는 *검색 품질*(coverage·recall·MRR)만 잰다. 그 1등을 챗봇에 적용해야 실제 답변이 좋아진다.
- **[오늘의 발견]** 지금 `app.py`는 `search()`(vector 전용) → 종합질문에서 정답 조항을 놓침. `retrieve(method="hybrid")`면 잡힘. **검색설정 = 답변품질(30점)의 병목.**
- 프롬프트: 오늘 놀이터 실험에서 **B(균형: 친절+간결+그라운딩)** 이 현재보다 짧으면서 더 명확·완전(품질↑·비용↓). C(따뜻+상세)는 3배 비싸고 출처밖 상식 섞임(그라운딩 위험).

## 목표

1. 리더보드로 **1등(+2등 finalist) 검색설정 확정**.
2. `app.py`가 그 설정(인덱스·검색방식·K·모델)으로 **검색**하게.
3. `app.py` SYSTEM_PROMPT를 **균형톤**으로.
4. **연습문제 5개** + holdout `split:"final"` 3~4개로 눈검증.

## 변경 대상 파일

### 수정
- `rag-starter/indexer.py` — (필요 시) 1등 임베딩으로 재색인해 `index.pkl` 교체.
- `rag-starter/backend/app.py` — `search()` → `harness.retrieval.retrieve(method,k,embed_model)`; SYSTEM_PROMPT 균형톤.

> ⚠️ **동시 세션 주의**: `06_30_chat_ui.md` 세션도 `app.py`를 건드림(meta 추가). **SYSTEM_PROMPT·retrieve 교체는 그 세션과 겹치지 않게** — 먼저 조율하고, 겹치면 한 세션이 몰아서. 코드는 즉시 커밋.

## 중요 규칙

- **품질만 보지 말고 비용도**: coverage 동률/근소차(12문제서 1문제≈8%)면 *더 싸고 단순한* 설정([수정4]③). K↑·큰모델·긴청크 = 입력토큰↑ = 비용↑.
- **finalist 2개**([수정3]): 1등만 믿지 말고 상위 2개를 생성으로 태워 최종 결정.
- 각 단계 끝: 검증 → `/simplify` → 커밋.

## 실행계획

### [ ] 단계 1: 리더보드로 최고 설정 확정
- `harness/leaderboard.py`(또는 CLI 20줄)로 `runs.jsonl` → 설정별 평균 coverage·recall·MRR + 평균 입력토큰(비용 프록시) 순위표.
- 1등 + 2등(finalist) 기록. **통과 기준**: 순위표가 뜨고, 1·2등 설정이 명확.

### [ ] 단계 2: 1등 설정으로 인덱스 재빌드
- 1등이 현재(route+minilm)와 다르면 `build_index(chunker, embed_model)`로 재색인 후 `save_index`.
- **통과 기준**: `index.pkl`이 1등 설정으로 교체, `load_index()[0]`에 `section`·`part` 존재.

### [ ] 단계 3: app.py를 최고 설정으로 (검색 교체)
- `from indexer import search` → `from harness.retrieval import retrieve`; `/api/chat`에서 `retrieve(q, INDEX, method=<1등>, k=<1등>, embed_model=<1등>)`.
- **통과 기준**: 종합질문(H10류)이 이제 정답 조항을 물어와 실제로 답함(vector 때 실패 → 개선 확인). 기존 채팅 regression 없음.

### [ ] 단계 4: 프롬프트를 균형톤으로
- SYSTEM_PROMPT에 B(균형) 규칙 추가: *직답 한 줄 → 필요한 특정만 → 모든 관련 경우 커버, 군더더기 없이 → 전부 [n] 인용 → 출처 밖 금지.*
- **통과 기준**: `try_prompt.py`로 연습문제 몇 개 돌려 현재보다 명확·완전하되 토큰 안 폭증. 출처밖 상식 없음.

### [ ] 단계 5: 연습문제 5개 + final holdout 눈검증
- 채팅 UI(`localhost:5173`)에서 연습문제 5개 → 답+§인용 확인. holdout `split:"final"` 3~4개로 과적합 점검.
- **통과 기준**: 5문제 다 옳은 조항 인용 + 완전한 답. 범위밖 질문 거부.

**검증 통과 시 커밋**: `feat(app): ship best retrieval config + balanced answer prompt`
